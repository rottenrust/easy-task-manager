# easy-task-manager

Минимальный учебный CLI-менеджер задач на Python.

## Возможности

- `add` — добавить задачу
- `list` — показать список активных задач
- `done` — отметить задачу выполненной
- `delete` — удалить задачу
- хранение задач в `tasks.json`
- резервная копия `tasks.json.bak` перед каждой записью
- базовый клиент Telegram Bot API на стандартной библиотеке (`task_manager/telegram_client.py`)
- базовый обработчик команд `/start`, `/help`, `/add`, `/today`, `/list`, `/done`, `/archive`, `/delete`, `/undo`, `/clear_archive` и добавления обычным текстом с ограничением доступа по owner id (`task_manager/telegram_bot.py`)
- базовое логирование действий Telegram-обработчика в файл `task_manager.log`
- автоматическая миграция legacy-формата `tasks.json` (`done` / `status=completed`) в новую модель при чтении

## Требования

- Python 3.10+
- Без сторонних зависимостей

## Запуск

Из корня проекта:

```bash
python3 -m task_manager.cli add "Купить молоко"
python3 -m task_manager.cli list
python3 -m task_manager.cli done 1
python3 -m task_manager.cli delete 1
```

## Telegram Bot Token и Owner ID

Для локального запуска бота через CLI нужны две переменные окружения:
- `TELEGRAM_BOT_TOKEN` — токен бота от `@BotFather`
- `TELEGRAM_OWNER_ID` — ваш Telegram user id (целое число)

Универсальная команда запуска (подставьте свои значения):

```bash
cd /path/to/project && TELEGRAM_BOT_TOKEN="123456:ABCDEF" TELEGRAM_OWNER_ID="123456789" python3 -m task_manager.cli telegram
```

## Как создать Telegram-бота и запустить локально

1. Создайте бота через `@BotFather`:
   - отправьте `/newbot`;
   - задайте имя и `username` бота;
   - сохраните выданный `TELEGRAM_BOT_TOKEN`.
2. Узнайте свой Telegram user id (это будет `TELEGRAM_OWNER_ID`), например через `@userinfobot`.
3. Запустите бота через CLI (основной и поддерживаемый способ), подставив свой путь к проекту, токен и id:

```bash
cd /path/to/project && TELEGRAM_BOT_TOKEN="123456:ABCDEF" TELEGRAM_OWNER_ID="123456789" python3 -m task_manager.cli telegram
```

Бот получает апдейты через `getUpdates` (long polling) c `timeout=30` и ведет `offset` (`update_id + 1`) для подтверждения обработанных апдейтов.
Сетевой timeout HTTP-клиента отделен от `getUpdates timeout`; при `socket.timeout` polling не завершает процесс, а безопасно продолжает цикл.

Остановка запуска: `Ctrl+C` (в консоли выводится `Bot stopped.`).

## Восстановление данных из резервной копии

Перед каждой записью `tasks.json` приложение сохраняет предыдущую версию в `tasks.json.bak`.

Если `tasks.json` поврежден (невалидный JSON или неверная корневая структура), приложение:
- сохраняет поврежденный контент в `tasks.json.bak`;
- автоматически восстанавливает рабочий `tasks.json` c пустым списком (`[]`).

Для ручного восстановления из backup:

```bash
cp tasks.json.bak tasks.json
```

## Telegram команды `/start`, `/help`, `/add`, `/today`, `/list`, `/done`, `/archive`, `/delete`, `/undo`, `/clear_archive` и добавление обычным текстом

Команда `/start` уже реализована в базовом обработчике апдейтов.

Ожидаемый ответ владельцу:

```
Привет!
Я помогаю вести личный список задач.
```

Если сообщение приходит не от `TELEGRAM_OWNER_ID`, бот отправляет отказ в доступе.

Команда `/help` уже реализована и возвращает список доступных на текущем этапе команд:

```
Доступные команды:
/start — приветствие и краткое описание бота
/help — список доступных команд
/add <текст> — добавить новую задачу
/today — показать активные задачи
/list — показать активные задачи
/done <id> — завершить задачу
/archive — показать выполненные задачи
/delete <id> — удалить задачу
/undo [id] — восстановить завершенную задачу
/clear_archive — очистить архив
```

Команда `/add <текст>` добавляет задачу в список. Если текст не передан, бот возвращает подсказку по использованию:

```
Использование: /add <текст задачи>
```

Обычное текстовое сообщение без команды также добавляет задачу (аналогично `/add`).

Команды `/today` и `/list` возвращают список активных задач.
Если активных задач нет, бот отвечает:

```
Пока нет активных задач. Добавьте новую через /add <текст задачи>.
```

В непустом списке задачи выводятся в мобильном формате:

```
Активные задачи:
• #1 — Купить хлеб
• #2 — Позвонить маме
```

Команда `/done <id>` отмечает задачу выполненной (архивирует ее).
Если `id` не передан или некорректен, бот возвращает:

```
Использование: /done <id задачи>
```

Команда `/archive` показывает архив выполненных задач.
Если архив пуст, бот отвечает:

```
Архив пока пуст.
```

В непустом архиве:

```
Архив задач:
✓ #1 — Купить хлеб
```

Команда `/delete <id>` удаляет задачу. Для некорректного ввода:

```
Использование: /delete <id задачи>
```

Команда `/undo` восстанавливает последнюю завершенную задачу из архива, а `/undo <id>` — конкретную архивную задачу.
Если архив пуст:

```
В архиве нет задач для восстановления.
```

Команда `/clear_archive` очищает только архивные задачи и не затрагивает активные.
Если неизвестная slash-команда (`/foo`), бот возвращает:

```
Неизвестная команда. Используйте /help.
```

## Публичная версия `v0.4-public`

Для публикации на GitHub используется отдельный деперсонализированный snapshot в папке `v0.4-public`.

В `v0.4-public` файлы `task_manager.log`, `tasks.json`, `tasks.json.bak` хранятся в репозитории как **sample data only**:
- только демонстрационные записи и задачи;
- без реальных логов, приватных идентификаторов, токенов и локальных путей.

Ручной процесс обновления:
1. Внести и проверить изменения в рабочей версии проекта.
2. Перенести изменения в `v0.4-public`.
3. Вернуть `v0.4-public/task_manager.log`, `v0.4-public/tasks.json`, `v0.4-public/tasks.json.bak` к эталонным sample-значениям.
4. Запустить проверки из раздела `docs/public-release.md`.
5. Публиковать только подготовленную публичную ветку.

## Структура

- `task_manager/cli.py` — обработка CLI-команд
- `task_manager/service.py` — бизнес-логика задач (сервисный слой)
- `task_manager/storage.py` — загрузка/сохранение `tasks.json`
- `task_manager/telegram_client.py` — минимальный клиент Telegram Bot API (long polling и отправка сообщений)
- `task_manager/telegram_bot.py` — базовый обработчик Telegram-апдейтов и команд `/start`, `/help`, `/add`, `/today`, `/list`, `/done`, `/archive`, `/delete`, `/undo`, `/clear_archive`, а также добавления обычным текстом
- `task_manager/telegram_polling.py` — цикл long polling через `getUpdates` для локального запуска бота
- `tasks.json` — файл с задачами
- `user-ideas.md` — журнал пользовательских идей для дальнейшей проработки
